Yoja Architecture

Yoja is an AI Assistant with a built in RAG model for your Google Drive contents. While interacting with OpenAI during chat, Yoja provides tools to the LLM using the OpenAI Assistants API.

Built-in tools

Currently, Yoja has three built in tools:

  • search_question_in_db
  • search_question_in_db_return_more
  • list_of_files_for_given_question

The three tools above take 'question' as a parameter.

Customizable aspects

There are three customizable aspects of Yoja tools:

  • Ability to exclude any of the tools above
  • The description of the tool
  • The description of the parameter named 'question' to the tool

Usage

The customized prompts are stored in a Google Slides file and a new chat is initiated using the newly provided 'New Chat with Custom Prompts' button. The custom prompts setting is valid for that chat and the system will reset to the default prompts at the next new chat.

When you click on this button, a modal will open and you can browse your Google Drive or Dropbox contents and choose the Slides file containing the prompts.

Prompts

The format for spreadsheet documents are as follows:

  • The first sheet of the spreadsheet must be the default name Sheet1
  • Sheet1 must contain one row per tool
  • Each row must contain five columns. The columns are described below

Columns

The format for spreadsheet documents are as follows:

  • Column 1: Name of tool Must be one of search_question_in_db, search_question_in_db_return_more, list_of_files_for_given_question
  • Column 2: Description of tool This column is the description of the tool given to ChatGPT. It is a cutomizable prompt. For example, the search_question_in_db tool's default description is Search confidential and private information and return relevant passages for the given question or search and return relevant passages that provide details of the mentioned subject You can replace this with your own prompt.
  • Column 3: Name of first parameter to this tool This column is the name of the first parameter to this tool. At this point all tools included with Yoja take one single parameter named 'question'. Hence this column should contain question and it cannot be modified.
  • Column 4: Type of first parameter to this tool This column is the type of the first parameter to this tool. At this point all tools included with Yoja take one single parameter named 'question' of type 'string'. Hence this column should contain string and it cannot be modified.
  • Column 5: Description of the first parameter to this tool This column contains the description of the first parameter to this tool. This is a customizable prompt. For example, the parameter question to the tool search_question_in_db is described by default as The question for which passages need to be searchedYou can replace this with your own description of the parameter.
